# greeting-generator

## 目标

- 本项目旨在开发一个符合 Open Agent Skills 规范的 Skill 包。Skill 包的发布目录位于项目内的 `greeting-generator` 子目录。
- 本项目的产物 $greeting-generator，是一个旨在指导 Agent 根据用户需求生成优雅、得体、自然且带有用户自身特征的、非套路化的节日祝福语。

## Skill 输入

- 本 Skill 采用交互式执行流程，所有输入应由 Agent 在和用户的交互式问答中引导用户进行输入。
- 当然，这并不意味着本 Skill 仅接受 prompt 中直接给出的简单输入。用户在某些步骤可以通过输入文件路径的方式，指示 Agent 通过读取文件获得输入。
- 本 Skill 所需的输入信息大致可分为以下几类：
  * 节日上下文信息：用户至少需要给出节日名称，也可补充节日背景、节日习俗、节日的常见祝福语等信息。
  * 用户画像：用户应向 Agent 介绍自己，包括自己的姓名、身份、职业、语言风格、性格特征等信息，以便 Agent 建立用户画像，为用户定制出符合自身特征的祝福语。
  * 祝福语应用场景及规格：用户应告知 Agent 所需祝福语的用途，例如 短信 / 微博 / 朋友圈 / 电子邮件 / 口头祝福，以及所需祝福语的大致长度。用户还可以告知 Agent 需要几条候选祝福语。若用户未显式告知 Agent，则应该生成 **3 条**候选祝福语。
  * 祝福语发送对象及信息：用户应告知 Agent 生成的祝福语的发送对象及相关信息，包括对方的姓名、称谓、性别、年龄（不一定需要精确）、与用户的关系、需要关注的问题等。用户可以通过上传文件的方式一次性发送多个对象的信息，Agent 应分别为每一位对象生成祝福语。

## Skill 输出

- 本 Skill 应根据用户给出的节日上下文、用户画像、祝福语应用场景及规格，为用户提供的**每一位发送对象分别生成符合用户要求数量的候选祝福语**。例如，用户输入了3个发送对象，要求生成4条候选祝福语，则 Agent 给出的总条数即为 3 × 4 = 12 条。
- 输出的祝福语应**按发送对象分组**，**采用 Markdown 格式输出**。
- 属于同一位发送对象的不同候选祝福语之间**应保证一定的风格或内容差异**，避免为同一发送对象生成多条同质祝福语。

## Skill 执行流程

1. **搜集节日信息**：首先，**必须**获取节日名称。若用户在初始 prompt 中未提供，则必须询问用户。随后，如果 Agent 拥有网络搜索能力，Agent 应通过网络搜索补充更多的节日上下文信息。Agent 也可以通过交互式问答的方式尝试从用户那里获取节日信息。当 Agent 认为收集到足够的上下文信息后，便可以进入下一步。
2. **建立用户画像**：Agent 应请求用户介绍自己的基本信息。**Agent 应提示用户自我介绍是可选的，用户提供的信息越详尽，生成的祝福语质量便可能越高、越贴合用户的需求。**如果用户因为担忧隐私泄露等原因拒绝回答，不必强求，只需提示用户生成的祝福语的定制性可能会降低。
3. **明确祝福语的用途和规格**：Agent 应询问用户祝福语的应用场景以及每条祝福语的大致长度。如果用户拒绝回答，Agent 可以自行推断并决策。随后 Agent 还应询问对于每个发送对象需要几条祝福语候选，默认为3条。
4. **提示用户提供发送对象信息**：询问用户祝福语的发送对象是谁，并提示用户最好提供以下信息，重要性由高到低排列：
  - 称谓
  - 发送对象与用户的关系
  - 年龄
  - 性别
  - 需要关注的问题
  - 姓名
Agent 应提示用户，信息越详尽，祝福语的质量可能就越高。
Agent 还应提示用户可以通过给出文件路径的方法来批量提供发送对象信息，并且提示用户目标文件需要满足以下要求：
  * 必须是可解析的文本文件或结构化数据文件，例如 Markdown | JSON | YAML | txt 等。
  * 推荐采用结构化数据以降低误解析风险。
  * 用户应至少提供称谓和发送对象与用户的关系，否则可以简单地通过“佚名”的方式生成通用祝福语
  * 推荐采用以下形式的输入文件：
```yaml
- 王老五:
  称谓: 王老师
  关系: 初中班主任
  年龄: 56
  性别: 女
  需要注意的点: 身体不好
- 张老六:
  称谓: 张老板
  关系: 客户
  年龄: 48
  性别: 男
  需要注意的点: 喜欢听恭维的话
- 佚名:
- 佚名:
```

5. **总结上下文信息，生成提示语**：生成的提示语需要采用以下格式输出：

```markdown
---
festival_name: 春节
festival_info: 公历2026，农历丙午马年
user: 李四
identity: 教授，工程师，理工科，性格直率豪爽
usage: 短信
length: 1~2句话
candidates: 3
---

# 王老五

- 敬爱的王老师：值此新春佳节之际，李四祝您丙午马年龙马精神，身体康健顺遂！愿新的一年里，暖阳常伴，万事顺心，慢享生活之美。
- ...
- ...

# 张老六

- 张总，李四祝您在丙午马年马到功成，事业如火如荼、大展宏图！愿您步步领先，财源广进，继续引领行业风骚，新年大吉！
- ...
- ...
```

6. 询问用户是否需要将结果写入文件，以及写入目标文件名是什么。目标文件名默认可以提示为当前目录下的一个带skill名称和时间戳的 .md 文件，若用户无异议则安默认名称写入。
